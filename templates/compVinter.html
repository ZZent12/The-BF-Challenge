The differences between Compiled programs VS Interpreted programs

If you are confused about the difference between a program (or programming language) being compiled or interpreted, that is fine because it is blurry if you do not normally work in computer science. The main difference between the two is that the compiled program will become an executable (generally a file with the .exe extension) that you run to get your desired output, while the interpreted program will be fed into another program which will spit back out the desired output

From the graphic above it might seem like interpreted programs are just flat out better because it is simpler, you just put the program into an interpreter and you get the output, no having to make a new file and running that to get the output. The truth is that each way has their pros and cons and is sometimes better than the alternative

COMPILED
PROS

The program will generally run faster than interpreted programs

You do not need the compiler to run the program after it has been made into an executable

The program will generally be smaller than interpreted programs

CONS

It takes time to compile the program

The executable will only work on the operating system that it was made on

It is generally harder to debug the program

BETTER USED IN

platform specific applications

applications that needs to run fast

INTERPRETED
PROS

The interpreter does not care about the operating system

It is easier to debug and test the program

You do not need to recompile the program for a change to go into effect

CONS

It will generally run slower than compiled programs

You need the interpreter to run the program

file sizes are generally bigger than compiled programs

BETTER USED IN

multi platform appiclations

applications that have changing requierments

I hope this has helped define the diffrence between these two things.
